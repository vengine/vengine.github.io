Detailed Description

template<typename T, typename INC_CALLBACK = FIncCallbackProc<T>, typename DEST_CALLBACK = FDestCallbackProc<T>, typename GARBAGE_COLLECTOR = FGarbageCollectProc<T>>
class xhn::SmartPtr< T, INC_CALLBACK, DEST_CALLBACK, GARBAGE_COLLECTOR >

一个RefObject的派生类，直接VNEW出来以后它的ref_count是0，这是必须将该对象赋值给SmartPtr。这样保存起来 如果该对象没有赋值给SmartPtr，而是作为参数传递给一个参数类型是SmartPtr的函数，那么， 该类转为SmartPtr参数时ref_count为1，函数返回时ref_count为0，就直接销毁对象了 这可能会引起一个bug
